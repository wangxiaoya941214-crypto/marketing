# 发版验收单

## 1. 当前发版口径

- 当前唯一生产口径：**Railway**
- 当前线上服务：`superev-marketing-assistant`
- 当前公网域名：`https://superev-marketing-assistant-production.up.railway.app`
- Vercel 只保留为兼容/预览链路，不作为本次发版评审和上线验收目标

---

## 2. 当前版本对外怎么定义

当前版本对外只定义为：

> **闭环分析工作台（首版）**

当前不要对外讲成：

- 经营驾驶舱正式版
- 完整闭环自动化平台
- 多表全自动打通成品

营销诊断继续保留为：

> **营销诊断（兼容模式）**

---

## 3. Go / No-go 放行标准

下面 8 项必须同时满足，才允许进入上线动作：

1. `npm run release:check` 通过
2. `npm run lint` 通过
3. `npm run build` 通过
4. `node --import tsx --test tests/*.test.ts` 通过
5. `npx playwright test tests/e2e/closed-loop-v2-entry.spec.ts` 通过
7. 当前生产平台口径唯一，且文档已统一为 Railway
8. 产品经理给出明确 `Go` 结论

任何一项不满足，结论都只能是：

> **No-go，禁止上线**

---

## 4. 角色签字点

### 前端

- 首页模式切换路径稳定
- 闭环四个页签切换后有稳定标题和主标识
- 旧模式回归不纳入本轮放行，留到下一版统一迭代

### 后端

- 旧模式 API 与闭环 API 共享实现口径一致
- 闭环导入、复核、快照、AI 状态字段稳定
- `release:check` 与白名单可真实代表候选版本

### UI

- `闭环分析 / 营销诊断 / 导入工作台 / 复核队列 / 经营驾驶舱` 命名统一
- 状态标签如 `主入口 / 兼容模式 / 待复核 / AI 已降级` 表达一致

### 产品经理

- 明确这版只叫“闭环分析工作台（首版）”
- 明确这版能讲什么、不能讲什么
- 给出最终 `Go / No-go`

---

## 5. 最小线上验收

上线前后至少各跑一遍：

1. 主线索表识别 -> 数据匹配页
2. 闭环底座导入 -> 复核队列 -> 闭环分析 -> 经营驾驶舱

当前版本先把第 2 条作为正式放行链路。  
第 1 条保留给下一版统一迭代时再一起验收。
