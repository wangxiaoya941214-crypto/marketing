# 产品需求文档（PRD）

## 1. 产品名称

SUPEREV 营销分析助手

## 2. 产品定义

SUPEREV 营销分析助手，是一个面向营销团队的智能分析工具。  
它把分散在文本、CSV、Word、图片、PDF 中的营销数据，统一识别并整理成标准结构，再通过规则分析引擎和 AI 报告生成能力，输出可复核、可复盘、可执行的营销诊断结果。

它不是单纯的聊天机器人，也不是纯表格录入工具。  
它更像一个“营销诊断工作台”，把下面这条链路打通：

```text
数据导入 -> 自动识别 -> 人工校对 -> 结构化分析 -> 结果展示 -> 导出汇报
```

## 3. 要解决的核心问题

### 3.1 数据整理耗时

营销复盘前，团队通常要手动搬运数据、清口径、补字段，耗时长，且容易遗漏。

### 3.2 转化漏斗看不清

知道结果不好，但不知道问题出在留资、私域承接、高意向筛选还是成交环节。

### 3.3 复盘输出不统一

不同人复盘结构不同，结论风格不同，导致报告难比较、经验难沉淀。

### 3.4 决策靠经验，难快速动作

即使看到了问题，也缺少可执行的预算建议、内容建议和动作清单。

### 3.5 当前版本尚未解决的问题

以下问题是下一阶段重点，不属于当前版本的强能力：

- 历史记录沉淀
- 渠道级横向对比
- 销售个人转化率
- 实时监控与预警
- 真正 ROI/ROAS 口径分析

## 4. 目标用户

### 核心用户

- 营销运营
- 内容运营
- 私域运营
- 复盘负责人
- 部门负责人

### 次级用户

- 销售主管
- 增长负责人
- 需要阅读营销复盘结果的管理层

## 5. 核心使用场景

### 场景一：活动结束后快速复盘

用户导入本期数据，系统自动整理和识别，在 1 次校对后快速产出诊断报告。

### 场景二：投放/内容效果诊断

用户查看漏斗、内容排名、预算建议，快速判断该加量、停量还是调整私域承接。

### 场景三：向上汇报

用户在结果页查看分析并导出 PDF，用于内部同步和复盘汇报。

## 6. 当前版本核心功能

### 6.1 多格式数据输入

支持：

- 直接粘贴文本
- CSV
- TXT
- Markdown
- Word（`.docx`）
- PDF
- 图片

### 6.2 自动识别

系统会根据文件类型调用：

- 规则解析
- CSV 模板解析
- OpenAI / Gemini 文件识别

### 6.3 数据匹配页

识别完成后，用户可以修正：

- 统计周期
- 目标值
- CPS 红线
- 投放金额
- 漏斗数据
- 内容明细
- 异常说明
- 优秀案例链接
- 上期数据（用于环比）

### 6.4 结构化分析引擎

当前已输出：

- 分产品 KPI
- 漏斗分析
- 问题诊断
- 内容排名
- 预算建议
- 动作清单
- 放量建议
- 可靠性说明
- 完整长报告

### 6.5 结果展示

当前结果页已支持展示：

- KPI 高亮卡片
- 预算占比 vs 成交占比
- 分产品漏斗
- 可靠性说明
- 环比变化
- 完整分析报告
- 内容排名
- 预算建议
- 动作清单
- 放量建议

### 6.6 导出

支持将结果页导出为 PDF。

## 7. 当前版本不做什么

以下暂时不纳入当前版本目标：

- 登录/权限系统
- 多人协作编辑
- 数据库持久化历史
- 自动任务调度
- 渠道级多维看板
- 销售个人绩效分析
- 实时告警中心

## 8. 产品价值主张

### 对团队的直接价值

1. 把数据整理时间从人工搬运，压缩为自动识别 + 人工确认
2. 把“感觉哪里有问题”变成“明确知道哪一步掉人”
3. 把复盘结论从口头经验，变成结构化报告
4. 把建议从模糊判断，变成预算、内容、动作三个层面的执行建议

### 当前最强价值

当前版本最强的两项价值是：

- 数据整理效率
- 漏斗问题定位能力

## 9. 当前版本的产品边界判断

当前更准确的定位是：

**一个单次营销周期的智能诊断工具，而不是长期经营分析平台。**

它已经能把团队从“人工整理 + 靠感觉判断”推进到“半自动整理 + 结构化诊断”，
但还没有完成“历史沉淀 + 实时监控 + 多渠道经营对比”。

## 10. 下一阶段优先需求

### P1

- 历史记录与结果复用
- 结构化指标表展示
- `contentInsights` / `overallRatingLabel` 等未展示字段补齐
- 异常预警模块

### P2

- 渠道对比漏斗
- 动作追踪模块
- 历史趋势视图

### P3

- 渠道级 ROI / ROAS
- 销售个人转化率
- 自动周报/预警

## 11. 成功标准（当前阶段）

当前版本的成功标准不设为“经营驾驶舱”，而设为：

1. 用户能顺利导入并识别营销数据
2. 用户能在数据匹配页完成校对
3. 用户能看到结构化结果和建议
4. 用户能导出可汇报的结果页
5. 输出结果对复盘动作有实际指导意义

## 12. 当前版本的一句话描述

**SUPEREV 营销分析助手，是一个把营销数据从“导入和整理”一路推进到“诊断和建议”的智能复盘工具。**
